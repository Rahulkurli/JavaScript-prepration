Core Concepts
The fundamentals interviewers probe to test genuine understanding vs surface knowledge.
🔍
01Explain the JavaScript Event Loop in detail. How does it handle the call stack, Web APIs, and task queues?+
Hard

JavaScript is single-threaded — it has one call stack. The event loop constantly checks: "Is the call stack empty? If so, take the next task from the queue."

1. Synchronous code runs on the Call Stack.
2. Async operations (setTimeout, fetch) are handed to Web APIs (browser) or libuv (Node.js).
3. When complete, callbacks go to the Task Queue (macrotasks) or Microtask Queue (Promises).
4. The Event Loop drains the Microtask Queue completely after each task before picking the next macrotask.

// What's the output order?
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// Output: 1, 4, 3, 2
// Sync first → Microtasks → Macrotasks
02What is hoisting? How does it differ between var, let, const, and function declarations?+
Medium

Hoisting moves declarations to the top of their scope during compilation. Only declarations hoist — not initializations.

var: Hoisted to function scope, initialized as undefined. let/const: Hoisted to block scope but stay in the Temporal Dead Zone (TDZ) — accessing them throws ReferenceError. function declarations: Fully hoisted (name + body).

console.log(x); // undefined (var hoisted)
var x = 5;

console.log(y); // ReferenceError: TDZ
let y = 5;

greet(); // Works — function fully hoisted
function greet() { return 'hi'; }

sayBye(); // TypeError — arrow fn not hoisted
const sayBye = () => 'bye';
03Explain type coercion. What's the difference between == and ===? Give tricky examples.+
Hard

Type coercion is JavaScript's implicit conversion between types. === (strict) never coerces. == (loose) follows the Abstract Equality Comparison algorithm.

console.log(0 == '')           // true
console.log(0 == false)        // true
console.log(null == undefined) // true
console.log(null == 0)         // false !
console.log(NaN == NaN)        // false (NaN !== NaN)
console.log([] + [])           // ""
console.log([] + {})           // "[object Object]"
console.log([] == ![])         // true (!
04What is the difference between null, undefined, and undeclared?+
Easy

undefined: Declared but not assigned. Default for uninitialized vars, missing params, missing object keys.

null: Intentional absence of value, explicitly assigned. typeof null === 'object' is a famous JS bug.

undeclared: Never declared in any accessible scope. Accessing throws ReferenceError (except via typeof).

let a;
console.log(a);          // undefined
console.log(typeof a);   // "undefined"

let b = null;
console.log(typeof b);   // "object" (the bug)
console.log(b === null); // true (correct check)

console.log(typeof z);   // "undefined" (safe)
console.log(z);          // ReferenceError!
05How does 'this' work? How does it differ in arrow functions vs regular functions?+
Hard

In regular functions, this is determined by how the function is called (dynamic binding). In arrow functions, this is lexically bound — it inherits from the enclosing scope at definition time and cannot be overridden.

const obj = {
  name: 'JS',
  regular() { console.log(this.name); },  // 'JS'
  arrow: () => { console.log(this.name); }, // undefined
  delayed() {
    setTimeout(function() {
      console.log(this.name); // undefined (lost context)
    }, 100);
    setTimeout(() => {
      console.log(this.name); // 'JS' (arrow inherits)
    }, 100);
  }
};
06Explain call(), apply(), and bind(). When would you use each?+
Medium

All three explicitly set this. call(ctx, a, b) invokes immediately, individual args. apply(ctx, [a,b]) invokes immediately, array of args. bind(ctx, a) returns a new permanently-bound function — great for partial application and callbacks.

function greet(greeting, punct) {
  return `${greeting}, ${this.name}${punct}`;
}
const user = { name: 'Alice' };

greet.call(user, 'Hello', '!');    // "Hello, Alice!"
greet.apply(user, ['Hi', '?']);   // "Hi, Alice?"
const bound = greet.bind(user, 'Hey');
bound('.');                         // "Hey, Alice."
07What are JS data types? Explain pass-by-value vs pass-by-reference.+
Medium

Primitives (by value): string, number, bigint, boolean, undefined, null, symbol. Immutable and copied on assignment.

Objects (by reference): Objects, Arrays, Functions. Variables hold a memory address — mutations affect all references.

let a = 5, b = a; b = 10;
console.log(a); // 5 (unchanged)

const arr1 = [1,2,3];
const arr2 = arr1;
arr2.push(4);
console.log(arr1); // [1,2,3,4] (mutated!)

// Safe copies
const shallow = [...arr1];
const deep = structuredClone(arr1);
08What is the difference between deep copy and shallow copy? How do you deep clone?+
Medium

A shallow copy copies top-level properties — nested objects still share references. A deep copy recursively copies all levels.

const orig = { a: 1, b: { c: 2 } };
const s = { ...orig };
s.b.c = 99;
console.log(orig.b.c); // 99! (shared ref)

// Option 1: structuredClone (modern, recommended)
const d1 = structuredClone(orig);

// Option 2: JSON (loses functions/dates/undefined)
const d2 = JSON.parse(JSON.stringify(orig));

// Option 3: recursive (handles edge cases)
function deepClone(obj) {
  if (obj === null || typeof obj !== 'object') return obj;
  if (Array.isArray(obj)) return obj.map(deepClone);
  return Object.fromEntries(
    Object.entries(obj).map(([k,v]) => [k, deepClone(v)])
  );
}
// topic 02
Async & Promises
The most commonly asked and most commonly misunderstood part of JavaScript.
01What is a Promise? Explain the three states and Promise chaining.+
Medium

A Promise represents eventual completion or failure of an async operation. States: pending → fulfilled or rejected. Settled states are final.

const p = new Promise((resolve, reject) => {
  setTimeout(() => resolve('data'), 1000);
});

p.then(data => {
  console.log(data);    // 'data'
  return data + '!';   // chain returns new Promise
}).then(val => console.log(val)) // 'data!'
  .catch(err => console.error(err))
  .finally(() => console.log('done'));
02Compare Promise.all, Promise.allSettled, Promise.race, and Promise.any.+
Hard

Promise.all: All must fulfill. Rejects fast on first rejection. Promise.allSettled: Waits for all, never rejects, returns status objects. Promise.race: First to settle (resolve or reject) wins. Promise.any: First to fulfill wins; only rejects if all reject (AggregateError).

const p1 = Promise.resolve(1);
const p2 = Promise.reject('err');
const p3 = Promise.resolve(3);

await Promise.all([p1,p2,p3]);        // throws 'err'
await Promise.allSettled([p1,p2,p3]); // [{ok,1},{err},{ok,3}]
await Promise.race([p1,p2,p3]);       // 1
await Promise.any([p2,p1,p3]);        // 1 (first fulfilled)
03Explain async/await. Common pitfalls and error handling patterns.+
Medium

async/await is syntactic sugar over Promises. An async function always returns a Promise. await pauses until the Promise settles.

async function fetchUser(id) {
  try {
    const res = await fetch(`/api/user/${id}`);
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    return await res.json();
  } catch(err) {
    console.error(err); throw err;
  }
}

// Pitfall: sequential vs parallel
// SLOW: awaiting sequentially
const a = await fetchA();
const b = await fetchB();

// FAST: run in parallel
const [a, b] = await Promise.all([fetchA(), fetchB()]);
04What are microtasks vs macrotasks? Name examples of each.+
Hard

Macrotasks: setTimeout, setInterval, I/O callbacks, UI rendering. Run one per event loop iteration.
Microtasks: Promise callbacks, queueMicrotask, MutationObserver. The entire microtask queue drains after every macrotask — they can starve the event loop!

console.log('start');
setTimeout(() => console.log('macro'), 0);
Promise.resolve()
  .then(() => console.log('micro 1'))
  .then(() => console.log('micro 2'));
console.log('end');
// start → end → micro 1 → micro 2 → macro

Closures & Scope
Deep understanding of closures separates intermediate from senior engineers.
01What is a closure? Why is it useful? Give real-world examples.+
Hard

A closure is a function that remembers variables from its outer scope even after that scope has returned. Functions in JS maintain a reference to their lexical environment.

// Counter factory (private state)
function makeCounter(start = 0) {
  let count = start;
  return {
    increment: () => ++count,
    decrement: () => --count,
    value: () => count
  };
}
const c = makeCounter(10);
c.increment(); // 11 — count inaccessible from outside

// Memoization via closure
function memoize(fn) {
  const cache = new Map();
  return (...args) => {
    const key = JSON.stringify(args);
    if (!cache.has(key)) cache.set(key, fn(...args));
    return cache.get(key);
  };
}
02The classic loop-closure bug: explain what goes wrong and 3 ways to fix it.+
Hard

With var, all closures in a loop share the same variable. By the time callbacks execute, i is the final loop value.

// Bug with var
for (var i = 0; i < 3; i++)
  setTimeout(() => console.log(i), 100);
// Prints: 3, 3, 3

// Fix 1: let (block scoped per iteration)
for (let i = 0; i < 3; i++)
  setTimeout(() => console.log(i), 100);
// Prints: 0, 1, 2

// Fix 2: IIFE to capture value
for (var i = 0; i < 3; i++)
  ((j) => setTimeout(() => console.log(j), 100))(i);

// Fix 3: bind
for (var i = 0; i < 3; i++)
  setTimeout(console.log.bind(null, i), 100);
03What is currying? Implement a curry function from scratch.+
Hard

Currying transforms f(a,b,c) into f(a)(b)(c). Each partial call returns a new function that remembers previous arguments via closure, until all args are provided.

function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn.apply(this, args);
    }
    return (...args2) => curried(...args, ...args2);
  };
}

const add = curry((a, b, c) => a + b + c);
add(1)(2)(3);   // 6
add(1, 2)(3); // 6
add(1)(2, 3); // 6

// Real use: reusable factories
const multiply = curry((a, b) => a * b);
const double = multiply(2); // partial application
double(5); // 10

Prototypes & OOP
JavaScript's prototype-based inheritance model — often asked but poorly understood.
01Explain the prototype chain. How does property lookup work?+
Hard

Every object has an internal [[Prototype]] link. Property access walks the chain: object → its prototype → ... → null. First match wins.

const animal = { breathes: true };
const dog = Object.create(animal);
dog.barks = true;

dog.barks;    // true (own property)
dog.breathes; // true (from animal prototype)
dog.flies;    // undefined (end of chain)

// Chain: dog → animal → Object.prototype → null

dog.hasOwnProperty('barks');    // true
dog.hasOwnProperty('breathes'); // false
'breathes' in dog;              // true (checks chain)
02Implement your own 'new' keyword. What does it actually do under the hood?+
Hard

The new keyword: (1) creates a new object, (2) sets its [[Prototype]] to constructor.prototype, (3) calls constructor with this = new object, (4) returns the object (or whatever object the constructor returns).

function myNew(Constructor, ...args) {
  // 1. Create with correct prototype
  const obj = Object.create(Constructor.prototype);
  // 2. Call constructor with obj as this
  const result = Constructor.apply(obj, args);
  // 3. Return result if object, else obj
  return (result && typeof result === 'object') ? result : obj;
}

function Person(name) { this.name = name; }
Person.prototype.greet = function() { return `Hi, ${this.name}`; };

const p = myNew(Person, 'Alice');
p.greet(); // "Hi, Alice"
03ES6 Classes vs prototypal inheritance — what are the actual differences?+
Medium

ES6 classes are syntactic sugar over prototypes — same mechanism underneath. Key differences: must use new, run in strict mode, not hoisted (TDZ), methods are non-enumerable by default, must call super() before using this in subclass constructor.

// ES6 Class
class Animal {
  constructor(name) { this.name = name; }
  speak() { return `${this.name} makes a sound`; }
}
class Dog extends Animal {
  speak() { return `${this.name} barks`; }
}

// Equivalent (manual prototype)
function Animal(n) { this.name = n; }
Animal.prototype.speak = function() { ... };
function Dog(n) { Animal.call(this, n); }
Dog.prototype = Object.create(Animal.prototype);
Dog.prototype.constructor = Dog;

Advanced Topics
Questions that signal you've gone beyond tutorials and worked in production.
01What are generators? How do they enable lazy evaluation?+
Hard

Generators are functions that can pause and resume using yield. They return an iterator. Perfect for infinite sequences, streaming data, and custom iterables.

function* fibonacci() {
  let [a, b] = [0, 1];
  while (true) {
    yield a;
    [a, b] = [b, a + b];
  }
}
const fib = fibonacci();
[...Array(8)].map(() => fib.next().value);
// [0,1,1,2,3,5,8,13] — computed lazily, no array needed

// Two-way communication
function* calc() {
  const x = yield 'Enter a number';
  const y = yield 'Enter another';
  return x + y;
}
const g = calc();
g.next();       // {value: 'Enter a number', done: false}
g.next(10);    // {value: 'Enter another', done: false}
g.next(20);    // {value: 30, done: true}
02Explain JavaScript's Proxy object. What can you build with it?+
Hard

Proxy wraps an object and intercepts fundamental operations via traps. Powers Vue 3's reactivity system, validation libraries, and mock objects.

// Validation proxy
const validate = (target, rules) => new Proxy(target, {
  set(obj, prop, value) {
    if (rules[prop] && !rules[prop](value))
      throw new TypeError(`Invalid: ${prop} = ${value}`);
    obj[prop] = value;
    return true;
  }
});

const user = validate({}, {
  age: v => typeof v === 'number' && v > 0 && v < 150
});
user.age = 25;     // OK
user.age = 'old';  // TypeError!

// Reactive data (Vue 3 core idea)
const reactive = obj => new Proxy(obj, {
  set(t, p, v) { t[p]=v; render(); return true; }
});
03What are WeakMap and WeakSet? When do you use them?+
Hard

WeakMap/WeakSet hold weak references — if no other reference exists, GC can collect the object (preventing memory leaks). Keys must be objects, non-iterable, no .size.

// Private instance data (no class private fields)
const _priv = new WeakMap();
class Secure {
  constructor(secret) { _priv.set(this, { secret }); }
  verify(s) { return _priv.get(this).secret === s; }
}
// secret inaccessible from outside

// Cache without memory leaks
const cache = new WeakMap();
function process(obj) {
  if (cache.has(obj)) return cache.get(obj);
  const result = /* expensive computation */;
  cache.set(obj, result);
  return result;
  // obj GC'd → entry auto-removed. No leak!
}
04Implement debounce and throttle from scratch.+
Hard

Debounce: Execute only after a quiet period (search inputs). Throttle: Execute at most once per interval (scroll/resize).

function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

function throttle(fn, limit) {
  let inThrottle = false;
  return function(...args) {
    if (!inThrottle) {
      fn.apply(this, args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}

const search = debounce(fetchResults, 300);
const onScroll = throttle(updateUI, 100);
05What causes memory leaks in JavaScript? How do you detect and prevent them?+
Hard

Common causes: forgotten event listeners, closures holding large data, detached DOM nodes, intervals not cleared, global variables accumulating state.

// LEAK: event listener never removed
function init() {
  const data = new Array(1000000).fill('x');
  document.addEventListener('click', () => {
    use(data); // data never freed!
  });
}

// FIX: AbortController
const ctrl = new AbortController();
document.addEventListener('click', handler, { signal: ctrl.signal });
ctrl.abort(); // removes listener

// LEAK: interval not cleared
const id = setInterval(poll, 1000);
// FIX:
clearInterval(id); // on cleanup

// Detect: Chrome DevTools → Memory tab → heap snapshot

What does this code output?
var x = 1;
function test() {
  console.log(x);
  var x = 2;
  console.log(x);
}
test();
1, 2
undefined, 2
1, 1
ReferenceError
Answer: undefined, 2. var x inside the function is hoisted to function scope, initialized as undefined. The outer x is shadowed. This is exactly why let/const are preferred.
What is logged?
const obj = { a: 1, b: { c: 2 } };
const copy = { ...obj };
copy.a = 99;
copy.b.c = 99;
console.log(obj.a, obj.b.c);
99, 99
1, 99
1, 2
99, 2
Answer: 1, 99. Spread creates a shallow copy. Primitive obj.a is copied by value (unchanged). But obj.b is copied by reference — mutating copy.b.c also mutates the original obj.b.c.
What does this print?
async function main() {
  console.log('A');
  await Promise.resolve();
  console.log('B');
}
main();
console.log('C');
A, B, C
A, C, B
C, A, B
A, C (B never runs)
Answer: A, C, B. main() starts synchronously, prints A. await suspends main and returns control. C prints. Then the microtask queue runs and B prints.
What is the output?
console.log(typeof null);
console.log(null instanceof Object);
"null", true
"object", true
"object", false
"null", false
Answer: "object", false. typeof null === 'object' is a historical V8 bug. But null instanceof Object is false because instanceof checks the prototype chain, and null has none.
What is logged?
function foo() {
  return { bar: 'hello' };
}
function baz() {
  return
  { bar: 'hello' };
}
console.log(foo());
console.log(baz());
{bar:'hello'}, {bar:'hello'}
{bar:'hello'}, undefined
SyntaxError in baz
{bar:'hello'}, null
Answer: {bar:'hello'}, undefined. ASI (Automatic Semicolon Insertion) inserts a semicolon after return in baz because there's a newline. The object literal becomes dead code. Always put { on the same line as return!
